WHy use GitHub 2.7

Features of the latest version of GitHub.

Git worktree

Git 2.7 has been released with some great new features, for example the new git worktree command, which allows you to check out and simultaneously work on multiple repository branches in separate directories. For example, if you need to make a fix and do not want to break your working copy you can check out a new branch in a new directory. This can be achieved with the following command:

$ git worktree add -b somedir/BB-1234 ../somedir/BB-1234

It is also possible to view the working tree as list alongside the working branches.

$ git worktree list
/Users/kannonboy/src/bb/bitbucket 37732bd [master]
/Users/kannonboy/src/bb/staging d5924bc [staging]
/Users/kannonboy/src/bb/fix/BB-1234 37732bd [hotfix/BB-1234]

Git rebase

Since Git 1.8.4 it has been possible to run a git rebase command and use the --autostash flag. So in affect this stashes any changes you may have made to the working copy before the rebase occurs, then reapplies them after the rebase. Git 2.7 provides us with a --no-autostash, that can be used like so.

$ git rebase master --no-autostash
