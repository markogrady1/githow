History of Git

How this form of version control came in to existence.

In April 2005, the Linux Kernal community were using bitkeeper as their primary source of version control. However, it could no longer meet their requirements. To solve this problem, Git founder Linus Torvalds took it upon himself to create a new type of version control. Linus disapeared over the weekend to emerge the following week with Git. This marked the moment when software development process would change forever.

Today Git is used for millions of projects around the world. To be exact 14 million people use GitHub as their central hub, while contributing to over 35 million projects. Now imagine how many people do not use GitHub as their central sharing point, but another technique such as a private hub. The mind boggles.

The Git communtity is thriving with features such mailing lists, bug reporting, an IRC channel and documentation. The IRC channel can be found at irc.freenode.net. Questions or comments for the Git community can be sent to the mailing list by using the email address a git@vger.kernel.org.

Contributing to Git

The Git documentation directory contains a repository full of files that explain features such as, rebasing from an internal branch, reverting a faulty merge, and using Git daemons. This repository will be of interest to developers who are looking to help or contribute. It also contains coding guidelines which promote the correct way to submit patches and for the true hardcore fan, there also consists a HowTo directory. Enjoy.
